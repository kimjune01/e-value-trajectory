# Data sources

## Real A/B test data

### Netflix sequential testing
- **Paper**: "Sequential A/B Testing Keeps the World Streaming" (2023)
- **URL**: https://netflixtechblog.com/sequential-a-b-testing-keeps-the-world-streaming-netflix-part-1-continuous-data-cba6c7ed49df
- **What**: Real Netflix A/B test trajectories with anytime-valid confidence sequences
- **Status**: Published figures, no raw data. May need to digitize plots or request data.

### Adobe anytime-valid inference
- **Paper**: "Anytime-Valid Confidence Sequences in an Enterprise A/B Testing Platform" (2023)
- **URL**: https://arxiv.org/pdf/2302.10108
- **What**: Thousands of real Adobe experiments with trajectory visualizations
- **Status**: Paper with plots, no public dataset.

### Johari et al. — Always Valid Inference
- **Paper**: "Always Valid Inference: Bringing Sequential Analysis to A/B Testing" (2021, Operations Research)
- **URL**: https://arxiv.org/abs/1512.04922
- **What**: Real A/B test data with confidence sequences evolving over time
- **Status**: Paper with examples.

## Benchmark / simulation data we generate ourselves

### Synthetic cyclic system
Generate data from a known feedback system (e.g., predator-prey, damped harmonic oscillator, logistic map). Compute e-values sequentially. Compare trajectory shape to known system dynamics.

### Synthetic stationary system
Generate i.i.d. data with known effect size. Compute e-values. Baseline: trajectory should diverge smoothly (effect present) or converge smoothly (null true). No oscillation.

### Synthetic regime-switching
Generate data that switches between two DGPs mid-stream (effect present → absent → present). Compute e-values. The trajectory should show regime changes visible in the evidence before any test detects them.
